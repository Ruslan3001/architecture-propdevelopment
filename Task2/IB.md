# Задание 2. Проверочный лист для бизнес-систем PropDevelopment

## Обоснование выбора разделов
1. **Управление доступом:** Вскрыты проблемы IDOR (подмена данных) при интеграциях с внешними Software System (`Поставщик ресурсов ЖКХ`). Пользователи `Клиент` и `Собственник` регистрируются в разрозненных БД, нет единого реестра.
2. **Безопасность данных:** Отсутствует системный контроль потоков к `Хранилище данных` [Greenplum]. Контракты с `Поставщик ресурсов ЖКХ` отдают избыточные ПДн, а на `Витрина сервисов для собственников` пользователи видят чужие данные.
3. **Защита инфраструктуры:** Доступ извне ограничивается только компонентом `Firewall`. Отсутствуют шлюзы (API Gateway) перед прикладными сервисами `tenant-core-app` и `client-mart-app`.
4. **Управление инцидентами:** Необходим системный подход к реагированию на инциденты и внедрение SIEM, так как текущие инциденты выявляются по жалобам через `Витрина продаж`.
5. **Интеграции и безопасность API:** Интеграция `tenant-core-app` с внешними системами (`Поставщик ресурсов ЖКХ`, `Платёжная система`) и `client-mart-app` с `Государственные регистрационные органы` является ключевым вектором утечки данных и компрометации сделок.
